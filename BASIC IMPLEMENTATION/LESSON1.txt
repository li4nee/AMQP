RABBIT MQ -> MESSAGE BROKER
RABBIT MQ-> USED SO THAT WE CAN SEND MESSAGES FROM ONE APPLICATION TO ANOTHER APPLICATION ASYNCHRONOUSLY
USED IN DISTRIBUTED SYSTEMS
USED IN MICROSERVICES

MESSAGE BROKER -> MIDDLEWARE BETWEEN TWO APPLICATIONS

PRODUCER ->  EXCHANGE -> QUEUE -> CONSUMER

EXCHANGE -> ROUTING KEY -> QUEUE

EXCHANGE KNOWS TO WHICH QUEUE TO SEND THE MESSAGE BASED ON THE ROUTING KEY.

# TYPES OF KEYS
1. BINDING KEY -> BINDS EXCHANGE WITH QUEUED / CONNECT EXCHANGE WITH QUEUE
2. ROUTING KEY -> IDENTIFIES THE QUEUE TO SEND DATA TO. USED BY EXCHANGE TO SEND DATA TO QUEUE


AMQP -> ADVANCED MESSAGE QUEUING PROTOCOL
